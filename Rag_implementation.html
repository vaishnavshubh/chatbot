RAG implementation — Financial Literacy Chatbot

This document describes the retrieval-augmented generation (RAG) layer: what was built, where it lives, how it affects the chatbot, and how to extend or disable it.

1. What problem RAG solves

Phase skills (md/skills/...) control behavior. RAG adds a curated knowledge base so Phase 3 — Evidence Intake and Phase 4 — Plan Generation can ground explanations in your team’s excerpts. RAG does not replace the orchestrator or analyzer; it augments the Speaker with extra system context.

2. Architecture

LayerRole
OrchestratorOwns phases and validation.
AnalyzerUnchanged; RAG does not feed the analyzer.
SpeakerOptional third system message: retrieved excerpts.
RAGDeterministic: load chunks → keyword ranking (topic-first) → top‑K → format text.

When RAG runs: In Phase 3 and Phase 4 when retrieved excerpts are attached to the Speaker. When it does not: RAG_ENABLED off, missing/empty chunks.jsonl, or retrieval error.

3. Key files

  • data/rag/*.md — source Markdown; filename = topic (matches primary_goal).
  • data/rag_index/chunks.jsonl — built index (JSONL).
  • app/rag/ingest.py — rebuild index.
  • app/rag/retrieval.py — RAGRetriever.
  • app/rag/prompts.py — format_rag_message().
  • app/orchestrator.py — attaches RAG context in Phase 3 and Phase 4.
  • app/speaker.py — rag_context parameter.

4. Retrieval

Keyword overlap scoring (no GPU). The retriever ranks on-topic chunks first and then backfills with other topics if needed. Query is built from primary_goal plus keyword hints in GOAL_KEYWORDS.

5. User-visible behavior

Phases 0–2 unchanged. Phase 3 evidence intake and Phase 4 plan generation are grounded on curated excerpts. Phase 5 unchanged unless you extend the code.

6. Environment

RAG_ENABLED=1
RAG_TOP_K=5

7. Rebuild index

python app/rag/ingest.py

8. Summary

Keyword RAG over JSONL; Phase 3 and Phase 4; disable via env; ingest regenerates chunks from data/rag/*.md.
